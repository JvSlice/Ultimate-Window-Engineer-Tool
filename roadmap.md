UWE Roadmap

Let engineers engineer faster—from one tool.

Purpose

This roadmap defines the planned direction for Ultimate Window Engineer (UWE). It is not a fixed promise or release schedule. It is a guide for development priorities, feature planning, GitHub Issues, and AI-assisted coding work.

UWE should grow as a practical engineering toolbox focused on fast design review, quick calculations, fabrication support, reference lookup, testing preparation, and real-world problem solving.

Development Philosophy

Every feature should answer this question:

Does this help an engineer solve a real-world problem faster, with more confidence, from one tool?

If the answer is yes, the feature is a strong candidate for UWE.

Current Core Areas

* Convert It
* Fabricate It
* Reference It
* Window Testing Tools
* Material Data
* Physics and engineering references
* Thermal calculations
* Glass and structural engineering utilities

Near-Term Priorities

1. Improve Search

Create a better search system that helps users quickly find tools, pages, references, and calculators without relying on manually maintained specific search keys.

Desired search data should include:

* Title
* Category or section
* Description
* Tags
* Aliases
* Route or destination page

2. Strengthen Navigation

Make it easier to move between major tool areas and related tools.

Focus areas:

* Clear page hierarchy
* Consistent back/navigation behavior
* Better grouping under Reference It, Physics, Testing, Fabrication, and Conversions

3. Expand Engineering Calculators

Continue adding practical calculators that support design review, fabrication, testing, and quick engineering checks.

Potential calculators:

* Thermal expansion
* Thermal stress
* Beam deflection
* Load and pressure tools
* Fastener and anchor checks
* Glass support calculations
* Buck and install condition calculators

4. Improve Material Data

Build a stronger shared material data system that can support multiple tools.

Material data may include:

* Coefficient of thermal expansion
* Density
* Modulus of elasticity
* Thermal conductivity
* Common engineering notes
* Default values used by calculators

5. Improve Testing Tools

Expand tools related to window and door testing.

Potential areas:

* ASTM E330 structural pressure references
* TAS test sequence support
* Water spray calculations
* Test setup checklists
* Pressure conversion helpers
* Report and field note utilities

Mid-Term Priorities

Favorites and Recently Used Tools

Allow users to quickly return to the tools they use most often.

Better Reference Pages

Create cleaner reference sections for engineering formulas, testing standards, materials, fabrication notes, and industry terminology.

Shared Calculator Framework

Create a consistent pattern for calculators so new tools can be added faster and more safely.

Each calculator should ideally have:

* Clear title
* Description
* Inputs
* Default values
* Formula explanation
* Output results
* Notes or limitations
* Search index entry

Documentation and Help

Add simple help text to tools so users understand what a calculator does, what assumptions it makes, and when to use it.

Long-Term Vision

UWE should become a one-stop engineering companion for the window and door manufacturing industry.

Long-term possibilities include:

* Offline-first reference database
* AI-assisted search and tool recommendations
* Engineering report generation
* User-defined calculators
* Project-based saved calculations
* Product testing workflows
* Fabrication planning tools
* Standards lookup and summary tools
* Desktop version for internal engineering use

Issue Planning

Future work should be tracked using GitHub Issues.

Each issue should include:

* Problem
* Goal
* Requirements
* Acceptance criteria
* Testing steps
* Related files or app areas

The first major issue for this roadmap should be:

Improve UWE Search
